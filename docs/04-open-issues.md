| # | 항목 | 슬라이드(규격) | 실제 구현 | 결정 필요 |
| --- | --- | --- | --- | --- |
| 1 | **주석 네임스페이스** | `male.sdi.dev/*` (슬라이드 3) | `male.keti.dev/*` | 둘 중 하나로 통일 |
| 2 | **A‑L‑E 값 의미** | 목표 지표값 (BLEU=26, latency 1200ms, energy‑class) | 0~1 가중치 | "가중치" vs "목표값" 통일, 혹은 병기 |
| 3 | **전달 채널** | GitOps(Argo CD) 유력 (슬라이드 6) | API Server HTTP 존재 | 최종 채널 확정 |
| 4 | **`score.top10`** (Cloud/Edge 후보) | 있음 (슬라이드 3) | 구현 미확인 | 사용 여부·포맷(`{cloud,edge,cloud-edge}`) 합의 |
| 5 | **CR vs 주석** 우선순위 | — | `MaleWorkload` CR + 직접 주석 둘 다 지원 | 표준 방식 권고안 결정 |

| 7 | **YAML 생성 라이브러리화** | KETI 담당 (슬라이드 6) | — | CR 변환/Helm 통합 라이브러리 범위 |

## 우선순위 제안

1. **(필수) #1 네임스페이스 · #2 값 의미** — 이게 안 맞으면 ETRI YAML이 그대로는 동작하지 않음.
2. **(필수) #3 전달 채널** — GitOps/HTTP 확정 후 파이프라인 구성.
3. (권장) #4 `score.top10`, #5 표준 방식.
4. (참고) #6 규칙표, #7 라이브러리화.
